初识GAS
   
1.1 什么是Game Ability System

虚幻引擎（Unreal Engine）中的Gameplay Ability System（GAS）是一个功能强大的技能系统框架，它主要用于管理游戏中的角色技能、属性、效果和交互。GAS系统通过一系列的类和接口，将角色的各种能力抽象为可重用的组件，从而简化了游戏逻辑的编写和维护。

1.1.1 Ability System Component

这是一种可以添加到角色的组件，它会处理许多重要的游戏系统中的事物，如技能学习，能力激活，激活能力时的通知处理。

1.1.2 Attribute Set

游戏技能系统（Gameplay Ability System）是使用游戏属性（FGameplayAttribute）来保存、计算和修改与角色相关的浮点值。这些浮点呢，可以描述拥有它的角色的任何特征，简单来说，就是把它挂载到你游戏主角、游戏AI等等，用来描述他们的生命值、耐力、还是说车辆的最高速度，或者说物品在损坏前可以使用的次数等等。游戏技能系统中的角色把他们的游戏属性存储再来一个属性集（AttributeSet）中，这个属性集有助于管理我们的游戏属性与系统其它部分之间的交互，并且把自己注册到角色的技能系统组件中。这些交互包括对数值范围的限定、临时数值变更、对永久改变基础值的事件做出反应等。

1.1.3 Gameplay Ability

它定义了游戏中节能功能的效果以及使用该技能所付出的代价（如果该技能有的话），以及什么时候再什么情况下可以使用该技能的相关信息等等。GA它是可以作为异步运行的实例化对象存在的，所以你可以运行专门的多阶段任务，包括角色动画、粒子和音效，甚至可以根据游戏执行时的玩家输入或者角色交互来设计技能分支。GA是可以在整个网络中自我复制的，运行在客户端或者服务器计算机上（甚至已经包括了客户端预测支持），还能同步变量和执行远程过程调用（RPC）。GA还可以在游戏中灵活实现游戏技能，比方说，提供的扩展功能是可用于实现冷却和使用消耗、玩家输入、使用动画蒙太奇的动画、以及对基于Actor的技能本身做出反应。

1.1.4 Ability Task

它是一种更常规的技能任务类的特殊形式，旨在使用游戏性技能。我们使用GAS的游戏通常是包括各种自定义任务的，这些任务都有它所独特的游戏功能。他们在游戏性技能执行过程中执行异步工作，并且可以调用委托（在本地C++代码中）或者经过一个/多个输出执行引脚（在蓝图中）来影响执行流。这使得我们的节能可以跨多个帧来进行，并且可以在同一帧内执行多个不同的函数。大部分技能任务都有一个即时触发的执行引脚，使蓝图能够在任务开始后继续执行。此外，通常还有一些特定于任务的引脚，它们会在延迟后或在可能发生或不发生的某个事件之后触发。
技能任务可以通过调用"EndTask"函数自行终止，或者等待运行它的游戏性技能结束，此时它会自动终止。这可以防止幻影技能任务运行，有效地泄漏CPU周期和内存。例如，某个技能任务可能播放一个施法动画，而另一个任务则在玩家的瞄准点处放置一个靶向标线。如果玩家点击确认输入来施放该法术，或者等待动画结束而未确认该法术，游戏性技能就会结束。虽然它们可以在任何时候自动终止，但是技能任务保证最晚在主要技能结束时终止。

1.1.5 Gameplay Effect

Gameplay技能系统利用Gameplay效果来更改Gameplay技能所针对Actor的属性。Gameplay效果包含了你可以应用到Actor属性的函数库。这些效果可以是即时效果，比如说施加伤害，也可以是一个持续的效果，比如说燃烧伤害，在一定时间内持续对玩家造成伤害。

1.1.6 Gameplay Cue

游戏提示部件。这个部件负责处理诸如粒子系统和声音等。

1.1.7 Gameplay Tag

这是GAS的另一个核心部件。现在Gameplay Tag实际上并不局限于GAS中。它还能用于非游戏项目的开发。Gameplay Tag的使用无处不在，贯穿整个游戏。对于识别几乎任何你能想到的事物都很有用。它的层次结构比枚举、布尔值或字符串等简单变量更加灵活。
